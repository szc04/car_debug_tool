🚗 车机自动化调试工具
一款专为车载 Android 系统开发的 串口 + ADB 双通道自动化调试工具，支持一键连接、命令执行、文件上传、设备重启及日志实时监控，大幅提升车机开发与测试效率。

✨ 核心功能
🔌 串口（UART）实时监控
自动连接指定串口（如 COM3），波特率可配置
实时滚动显示串口输出日志（线程安全，打包后仍正常工作）
支持发送任意串口命令（如 getprop, dmesg, reboot 等）
内置 中断按钮（模拟 Ctrl+C），可随时停止串口输出

📱 ADB 命令执行与日志捕获
自动识别已连接的 Android 车机设备
支持两类命令：
车机 Shell 命令：如 ls /system、cat /proc/version
本地 ADB 工具命令：如 adb pull、adb logcat -d > log.txt
日志实时显示并自动保存到 adb.log
📤 文件批量上传

支持同时上传 两个本地文件 到车机指定路径（如 /data/local/tmp/）
自动校验文件是否存在，避免无效操作

🔁 一键重启车机
在上传文件后自动执行重启命令（如 reboot）
适用于固件更新、服务重载等场景
🧪 分步式调试流程
工具将调试过程划分为 6 个标准步骤，左右双列布局，逻辑清晰：

步骤	功能描述
步骤1	连接串口 + 执行初始化命令（如读取系统属性）
步骤2	执行 ADB 命令（支持 Windows 本地命令）
步骤3	上传文件 + 重启车机
步骤4	车机启动后执行串口命令（带中断按钮）
步骤5	车机启动后执行串口命令（常规）
步骤6	执行 ADB 命令（如拉取日志、导出数据）

💾 配置持久化
所有设置（串口号、命令、文件路径等）自动保存到 debugger_config.json
下次启动自动加载，无需重复配置

📁 日志自动保存
串口日志 → serial.log
ADB 日志 → adb.log
每次更新覆盖写入，保留最近 500 行内容
🛠️ 使用要求
操作系统：Windows 7/10/11（需安装 Python 环境或使用打包版）
依赖硬件：
车机通过 USB 转串口模块连接 PC（如 CH340、CP2102）
车机开启 ADB 调试并授权 PC
权限：运行程序需有串口访问权限（部分系统需管理员权限）

📦 快速开始
方式一：直接运行（需 Python 环境）
bash
编辑
pip install pyserial adbutils
python car_debugger_pro.py
方式二：使用打包版（推荐）
下载编译好的 .exe 文件（由 PyInstaller 打包）
确保 debugger_config.json 与主程序在同一目录
双击运行即可，无需安装 Python
💡 首次使用请在"步骤1"中确认串口号（如 COM3）和波特率（如 115200）

🔒 线程安全设计（重要）
本工具采用 队列 + 主线程刷新机制，确保：
串口/ADB 后台线程 不直接操作 GUI
所有日志更新均在 Tkinter 主线程中完成
完美兼容 PyInstaller 打包后的 --onefile 模式
